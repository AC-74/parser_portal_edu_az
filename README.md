# Интерактивный каталог программ портала portal.edu.az

Этот проект обрабатывает локальные JSON-файлы с данными портала и создает единый интерактивный HTML-отчет для удобного анализа и фильтрации учебных программ.

## Установка и запуск

1.  **Клонируйте репозиторий и перейдите в папку проекта.**

2.  **Создайте виртуальное окружение и установите зависимости:**
    ```bash
    python -m venv .venv
    .venv\Scripts\activate
    pip install -r requirements.txt
    ```

3.  **Получите данные из браузера (см. инструкцию ниже) и поместите их в корень проекта.**
    *   `foreigner_specialities.json`
    *   `atis_enterprisers.json`

4.  **Запустите скрипт для генерации отчета:**
    ```bash
    .venv\Scripts\process-portal-data.exe
    ```
    По умолчанию отчет будет сохранен в файл `programs_report.html`.

## Расширенное использование

Вы можете использовать аргументы командной строки для фильтрации данных перед генерацией отчета.

*   **Фильтрация по цене:**
    ```bash
    # Установить минимальную и максимальную цену
    .venv\Scripts\process-portal-data.exe --min-price 1000 --max-price 3000
    ```

*   **Фильтрация по специальностям:**
    ```bash
    # Выбрать одну или несколько специальностей (через запятую, в кавычках)
    .venv\Scripts\process-portal-data.exe --specialties "Международные отношения,Юриспруденция"
    ```

*   **Получить список всех доступных специальностей:**
    ```bash
    .venv\Scripts\process-portal-data.exe --list-specialties
    ```

*   **Указать другой путь для сохранения отчета:**
    ```bash
    .venv\Scripts\process-portal-data.exe --output-html C:\path\to\your\report.html
    ```

## Как получить данные из браузера

Для работы скрипта требуются актуальные данные, которые можно получить вручную с сайта portal.edu.az. Вот как это сделать:

1.  **Откройте инструменты разработчика.**
    *   Зайдите на сайт [portal.edu.az](https://portal.edu.az/).
    *   Нажмите клавишу `F12` (или `Ctrl+Shift+I` / `Cmd+Option+I`) чтобы открыть инструменты разработчика.

2.  **Перейдите на вкладку "Сеть" (Network).**
    *   Внутри инструментов разработчика найдите и выберите вкладку "Network".

3.  **Отфильтруйте запросы.**
    *   Чтобы убрать лишнее, нажмите на фильтр `Fetch/XHR`. Это покажет только запросы к API, которые нас интересуют.

4.  **Найдите и сохраните нужные файлы.**
    *   Находясь на сайте, перейдите в раздел выбора специальностей для иностранцев (`Əcnəbi tələbələr üçün`).
    *   В панели Network вы увидите список запросов. Нас интересуют в первую очередь два:
        *   Запрос, содержащий в названии `foreigner_specialities`.
        *   Запрос, содержащий в названии `atis_enterprisers`.

5.  **Сохранение ответа в JSON-файл:**
    *   Нажмите на нужный запрос в списке.
    *   В открывшейся справа панели перейдите на вкладку **"Response"** (Ответ).
    *   Вы увидите текст в формате JSON. Скопируйте **всё** его содержимое (`Ctrl+A`, `Ctrl+C`).
    *   Вставьте скопированный текст в текстовый редактор и сохраните файл с соответствующим именем (`foreigner_specialities.json` или `atis_enterprisers.json`) в папку с проектом.

После того как эти два файла будут в папке, вы можете запускать скрипт для генерации отчета.

## Структура проекта и назначение файлов

*   `atis_enterprisers.json`: Исходные данные о предприятиях/университетах.
*   `foreigner_specialities.json`: Исходные данные о специальностях для иностранных студентов.
*   `university_details.json`: Дополнительные детали об университетах (адреса, координаты, описания, URL фото).
*   `full_translation_map.json`: Словарь для хранения переводов азербайджанских терминов на русский язык. Используется для динамического перевода в отчете.
*   `programs_report.html`: Сгенерированный интерактивный HTML-отчет с каталогом программ.
*   `requirements.txt`: Список Python-зависимостей проекта.
*   `pyproject.toml`: Файл конфигурации проекта (например, для Poetry).
*   `gemini.md`: Подробный план проекта, текущий статус, выполненные и оставшиеся задачи, а также известные проблемы.
*   `README.md`: Основная документация проекта, инструкции по установке, запуску и использованию.
*   `test_data_selection.py`: Вспомогательный скрипт для проверки корректности фильтрации данных из `foreigner_specialities.json`.
*   `images/placeholder.png`: Изображение-заполнитель, используемое в отчете, если фотография университета недоступна.
*   `src/portal_data_processor/`: Основной каталог с исходным кодом приложения.
    *   `src/portal_data_processor/cli.py`: Точка входа для командной строки, обрабатывает аргументы и запускает процесс генерации отчета.
    *   `src/portal_data_processor/client.py`: Содержит основную логику обработки, фильтрации и обогащения данных, а также класс `PortalDataClient`.
    *   `src/portal_data_processor/report_template.html`: Jinja2 шаблон для генерации HTML-отчета.
