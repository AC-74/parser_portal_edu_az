# План решения задачи по выгрузке данных с portal.edu.az (Версия 4)

## Цель

Создать многофункциональный офлайн-каталог в виде единого HTML-файла. Каталог позволит интерактивно исследовать магистерские программы, фильтровать их по цене и названию, а также предоставит обогащенную информацию о каждом университете (описание, фото, адрес, расстояние до центра).

## Этап 1: Определение релевантных университетов и сбор их данных (Оптимизированный подход)

1.  Скрипт **сначала** загрузит и обработает `foreigner_specialities.json`.
2.  Применит к нему основные фильтры: **Magistratura, Əyani, Rus dili**.
3.  Из отфильтрованного списка программ скрипт получит **уникальный список ID университетов**, по которым нам действительно нужна информация.
4.  **Только для этих** университетов я запущу веб-поиск для сбора адресов, координат, фото и описаний.
5.  Все найденные данные будут сохранены в `university_details.json` для кэширования.

## Этап 2: Полная переработка скрипта

1.  **Новая логика в `client.py`:**
    *   Скрипт будет загружать данные из трех источников: `foreigner_specialities.json`, `atis_enterprisers.json` и нового `university_details.json`.
    *   Все данные будут объединены в единую таблицу.
    *   Будет добавлена функция для расчета расстояния между двумя координатами (университет и станция метро "28 мая").
    *   Функция фильтрации будет расширена для поддержки диапазона цен и списка выбранных специальностей.
2.  **Новый `cli.py` с двухэтапным подходом:**
    *   **Команда `list-specialties`:** Новый режим работы скрипта, который просто прочитает данные, найдет все уникальные специальности и выведет их в консоль в формате `Название на русском (Оригинал)`, чтобы вы могли выбрать нужные.
    *   **Основная команда `generate-report`:** Будет принимать новые аргументы для фильтрации (`--min-price`, `--max-price`, `--specialties`) и генерировать итоговый HTML-отчет.
3.  **Создание продвинутого HTML-отчета:**
    *   **Карточная верстка:** Отчет будет сверстан в виде "карточек". Каждая карточка — это один университет.
    *   **Содержимое карточки:** Фото университета, название, краткое описание, адрес, расстояние до центра и таблица с его программами.
    *   **Интерактивность:** Таблицы внутри карточек можно будет сортировать.

## Этап 3: Создание документации

Я создам файл `README.md` и напишу в нем подробную инструкцию, как с помощью инструментов разработчика в браузере найти и сохранить нужные JSON-файлы с портала.
