# План решения задачи по выгрузке данных с portal.edu.az

## Цель

Создать многофункциональный офлайн-каталог в виде единого HTML-файла. Каталог позволит интерактивно исследовать магистерские программы, фильтровать их по цене и названию, а также предоставит обогащенную информацию о каждом университете (описание, фото, адрес, расстояние до центра).

## Текущий статус проекта

### Выполненные задачи:

1.  **Определение релевантных университетов и сбор их данных (частично)**:
    *   Скрипт загружает и обрабатывает `foreigner_specialities.json`.
    *   Применяет основные фильтры: **Magistratura, Əyani, Rus dili**.
    *   Из отфильтрованного списка программ скрипт получает уникальный список ID университетов.
    *   **Примечание**: Веб-поиск для сбора адресов, координат, фото и описаний для этих университетов был реализован, но столкнулся с проблемой неактуальных URL-адресов фотографий.
2.  **Полная переработка скрипта**:
    *   **Новая логика в `client.py`**:
        *   Скрипт загружает данные из трех источников: `foreigner_specialities.json`, `atis_enterprisers.json` и `university_details.json`.
        *   Все данные объединяются в единую таблицу.
        *   Добавлена функция для расчета расстояния между двумя координатами (университет и станция метро "28 мая").
        *   Функция фильтрации расширена для поддержки диапазона цен и списка выбранных специальностей.
        *   Код был переработан в объектно-ориентированный стиль с использованием класса `PortalDataClient`.
        *   Удалены жестко закодированные словари `UNI_MAP` и `SPEC_MAP`.
        *   Реализована загрузка переводов из `full_translation_map.json`.
        *   Исправлена логика формирования путей к JSON-файлам.
        *   Добавлена динамическая подстановка переводов для названий специальностей, названий университетов и описаний университетов.
    *   **Новый `cli.py` с двухэтапным подходом**:
        *   Команда `list-specialties`: читает данные, находит все уникальные специальности и выводит их в консоль в формате `Название на русском (Оригинал)`.
        *   Основная команда `generate-report`: принимает аргументы для фильтрации (`--min-price`, `--max-price`, `--specialties`) и генерирует итоговый HTML-отчет.
3.  **Создание продвинутого HTML-отчета**:
    *   Отчет сверстан в виде "карточек" (один университет - одна карточка).
    *   Содержимое карточки: фото университета, название, краткое описание, адрес, расстояние до центра и таблица с его программами.
    *   `University_ATIS_ID` удален из видимого HTML-вывода.
    *   Отображается: название программы, фото и краткое описание.
    *   Реализована логика для отображения общего изображения-заполнителя, если `Photo_URL` университета пуст или недействителен.
    *   Интерактивность: таблицы внутри карточек можно сортировать. Добавлены инструменты для поиска и определения диапазона (строка поиска, выпадающие списки для фильтрации по программе, ползунки для диапазона стоимости обучения). Реализована JavaScript-логика для динамической фильтрации и сортировки карточек университетов.
    *   Шаблон `report_template.html` был скорректирован для устранения дублирования названий специальностей в таблице программ. Теперь отображается одна колонка в формате "Русское название (Азербайджанское название)".
4.  **Проверка корректности выборки данных**:
    *   Создан и успешно выполнен тестовый скрипт `test_data_selection.py`, который подтвердил, что фильтрация данных (магистратура, очная форма, русский язык) работает корректно.

### Оставшиеся задачи и известные проблемы:

1.  **Неполные переводы**:
    *   Файл `full_translation_map.json` содержит много терминов, для которых отсутствуют русские переводы (значение равно ключу). Это приводит к тому, что в отчете отображаются азербайджанские названия вместо русских.
    *   **Необходимые действия**: Необходимо заполнить `full_translation_map.json` корректными русскими переводами для всех отсутствующих терминов. Это можно сделать вручную или с помощью веб-поиска для каждого термина.
2.  **Неработающие ссылки на фотографии**:
    *   Ссылки на фотографии университетов в файле `university_details.json` недействительны (возвращают ошибку 404). В отчете вместо фотографий отображаются изображения-заполнители.
    *   **Необходимые действия**:
        1.  Найти актуальные URL-адреса высококачественных фотографий для каждого университета, представленного в `university_details.json`.
        2.  Обновить файл `university_details.json`, заменив старые `photo_url` на новые, рабочие ссылки.
        3.  Перезапустить скрипт генерации отчета, чтобы загрузить новые изображения и обновить `programs_report.html`.
3.  **Реализовать фильтрацию по языку**:
    *   Добавить аргумент `--lang` в `cli.py` для выбора языка обучения (например, `en`, `ru`).
    *   Изменить логику фильтрации в `client.py` для использования этого аргумента.

## Структура данных

*   **`foreigner_specialities.json`**: Содержит информацию о магистерских программах. Каждый объект программы включает:
    *   `name`: Название специальности (на азербайджанском).
    *   `ATIS_ID`: Уникальный идентификатор специальности.
    *   `enterprise_ATIS_ID`: Идентификатор университета, к которому относится программа (связь с `university_details.json`).
    *   `educationLanguageId`: Идентификатор языка обучения.
    *   `entranceSpecialtyPaymentAmount`: Стоимость обучения.
    *   Другие поля, относящиеся к программе.

*   **`university_details.json`**: Содержит подробную информацию об университетах. Каждый объект университета включает:
    *   `ATIS_ID`: Уникальный идентификатор университета.
    *   `name`: Название университета (на азербайджанском, иногда на русском).
    *   `address`: Адрес университета.
    *   `latitude`, `longitude`: Координаты университета.
    *   `description`: Описание университета (на азербайджанском, иногда на русском).
    *   `photo_url`: URL фотографии университета.

*   **`atis_enterprisers.json`**: Содержит дополнительную информацию о предприятиях/университетах, используется для получения названий университетов.

### Взаимосвязи данных

*   `foreigner_specialities.json` связан с `university_details.json` через поле `enterprise_ATIS_ID` (в `foreigner_specialities.json`) и `ATIS_ID` (в `university_details.json`). Это позволяет связать каждую программу с соответствующим университетом и его деталями.

## Текущие проблемы и задачи

### 1. Неполные переводы

Файл `full_translation_map.json` содержит много терминов, для которых отсутствуют русские переводы (значение равно ключу). Это приводит к тому, что в отчете отображаются азербайджанские названия вместо русских.

**Необходимые действия**: Необходимо заполнить `full_translation_map.json` корректными русскими переводами для всех отсутствующих терминов.

### 2. Неработающие ссылки на фотографии

Ссылки на фотографии университетов в файле `university_details.json` недействительны (возвращают ошибку 404). В отчете вместо фотографий отображаются изображения-заполнители.

**Необходимые действия**:

1.  Найти актуальные URL-адреса высококачественных фотографий для каждого университета, представленного в `university_details.json`.
2.  Обновить файл `university_details.json`, заменив старые `photo_url` на новые, рабочие ссылки.
3.  Перезапустить скрипт генерации отчета, чтобы загрузить новые изображения и обновить `programs_report.html`.
